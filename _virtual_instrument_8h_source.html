143 // Initially all clump had the ability to wait on timers, now that has grown to
144 // timers and objects such as the queue. Yet in many cases clumps never to need to
145 // on anything. In the simple case of no waiting several pointers can be saved.
146 // So The clump is going to migrate from one fixed, to a collection of ObserverState objects
147 // These are the step
148 //
149 // 1. move from one hardcoded time list to a fixed set of of WaitableSates that can be used for
150 // timers AND synchronization objects.
151 //
152 // 2. Move the fixed set of buffers to a set that can be dynamically allocated (and arrays)
153 //
154 // 3. Get Arrays to support nullptr-means-empty, or shared empty instances
155 //
156 // 4. Can other users of the _next field use the same mechanism?
